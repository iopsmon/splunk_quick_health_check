Splunk Index Configuration Parameters

Parameter Description Simple Terms
maxDataSize The maximum size, in megabytes. Specifies that a hot bucket can reach before splunkd triggers a roll to warm
FrozenTimePeriodInSecs Time period, in seconds, after which data is considered "frozen." Determines how long data is retained in the index before becoming non-searchable.
homePath.maxDataSizeMB Maximum size of the home path for an index (in megabytes). Limits the size of the directory where raw data resides for an index.
coldPath.maxDataSizeMB Maximum size of the cold path for an index (in megabytes). Limits the size of the directory where frozen (non-searchable) data is stored.
maxHotBuckets Maximum number of hot buckets (actively being written to) allowed for an index. Limits the number of actively written buckets to avoid excessive resource usage.
maxHotSpanSecs Maximum time period (in seconds) for which a bucket is considered "hot. Specifies how long a bucket remains in the hot/warm state before moving to cold.
maxTotalDataSizeMB Total maximum size of an index, including both home path and cold path (in megabytes). Limits the overall size of the index, including raw and frozen data.
maxWarmDBCount Maximum number of warm buckets allowed for an index. Limits the number of warm buckets that can exist simultaneously.
